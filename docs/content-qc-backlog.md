# 内容细化、Python 校验、KaTeX 修复与内链建设 Backlog

本文件是后续内容质量治理的执行清单。默认执行顺序固定为：

1. 内容细化和数学校对
2. Python 代码检查与重写
3. KaTeX warning 修复
4. 自动内联链接

这样安排的原因是：前两步会持续修改正文与公式，先做可以避免 KaTeX 和内链阶段反复返工。

## 执行原则

- 每次只推进一个册或一个高相关专题，避免全站来回切换。
- 先修基础册和高复用术语，再处理后续书册。
- 每一阶段完成后都要重新运行格式化和构建检查。
- 不保留“能跑但没教学价值”的 Python 代码块。
- 自动内链先试点，再全站铺开。

## 统一问题标签

### 内容类

- `MATH-ERR`: 数学结论、答案或推导错误。
- `MATH-COND`: 定义域、参数范围、存在条件、边界情况漏写。
- `MATH-STEP`: 推导跳步，初学者难以自行补出。
- `MATH-NOTE`: 表述不严谨、符号不统一、语言易误解。
- `PEDA-GAP`: 动机、直观解释、方法总结、常见错误讲得不够。

### Python 类

- `PY-RUN`: 代码不可运行或高风险不可运行。
- `PY-FMT`: 代码格式、命名、输出不清晰。
- `PY-WEAK`: 代码只是打印字符串、公式或口号。
- `PY-MISMATCH`: Python 输出与答案或讲解不一致。
- `PY-MEANING`: 代码能运行，但没有验证或观察价值。

### KaTeX 类

- `KATEX-CN`: 数学模式里直接混入中文。
- `KATEX-UNIT`: 单位写法触发 warning 或风格混乱。
- `KATEX-TEXT`: 解释性文本塞进公式环境。
- `KATEX-MISC`: 其他渲染问题。

### 链接类

- `LINK-TERM`: 值得自动链接的术语。
- `LINK-ALIAS`: 需要别名支持的术语。
- `LINK-AMB`: 歧义术语，不应直接自动链接。
- `LINK-SKIP`: 应跳过自动链接的位置或词。

## 并行补充事项

这几项不必阻塞当前四大主线，但建议在主线推进过程中逐步补上：

1. 内容 QA 自动检查
   - 检查每页是否恰好 5 道练习。
   - 检查每题是否有答案、讲解、Python 三部分。
   - 检查 frontmatter、`order`、标题、描述是否齐全。
2. Python 编辑器性能优化
   - 后续考虑把高密度 `InlinePythonPlayground` 改成惰性激活。
3. 站内搜索
   - 目前页面体量已值得加入站内搜索。
4. 相关知识与前置知识区块
   - 与自动内链形成互补。
5. 多语言策略说明
   - 明确 UI 多语言与内容多语言的边界。

## 每轮执行完成后的固定检查

每处理完一个册或一个明显子阶段，都执行：

1. `npm run format`
2. `npm run build`
3. 抽样检查若干页面的公式、Python、分页与目录

如果后续需要更细粒度的执行记录，可以在本文件基础上继续追加“已完成 / 进行中 / 待处理”分区，而不另起一套标准。
