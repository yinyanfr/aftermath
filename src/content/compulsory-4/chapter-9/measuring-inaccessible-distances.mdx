# 数学探究：测量不可达距离

## 学习目标

学完这一页后，你应该能：

1. 为不可直接到达的两点设计测量方案；
2. 说明为什么需要基线、角度和模型假设；
3. 用正弦定理或余弦定理计算目标距离；
4. 分析误差来源；
5. 用表格整理一次完整的测量活动。

## 为什么要学这个内容

前面几页解决的是“给定数据怎样计算”，这一页要解决的是“数据从哪里来”。

数学建模不只是在纸上列公式，还包括：

1. 选择对象；
2. 设计方案；
3. 采集数据；
4. 检验结果。

## 预备知识

你需要会：

1. 正弦定理与余弦定理；
2. 基本测角含义；
3. 三角形内角和与面积公式。

## 一个标准测量思路

设不可到达的两点为 $A,B$。

常见思路是：

1. 在可到达区域选两个观测点 $C,D$；
2. 测量基线 $CD$；
3. 测量若干个角，如 $\angle ACD,\angle ADC,\angle BCD,\angle BDC$；
4. 先在 $\triangle ACD$ 和 $\triangle BCD$ 中分别求 $AC,BC$；
5. 再在 $\triangle ABC$ 中求目标距离 $AB$。

## 为什么这种方法可行

因为直接测 $AB$ 做不到，但只要能把问题拆成几个“已知一边和两个角”的三角形，就能把不可达边一步一步转化出来。

这正是解三角形在测量中的核心作用。

## 活动记录建议

一次完整活动至少要记录：

1. 小组成员与分工；
2. 测量对象；
3. 测量原理；
4. 使用工具；
5. 原始数据；
6. 计算过程；
7. 误差分析；
8. 结果对比与反思。

## 例题 1

如果只在一个观测点测得塔顶仰角，为什么通常还不能求出塔高？

### 讲解

因为塔高问题通常至少包含两个未知量：

1. 观测点到塔底的水平距离；
2. 塔的高度。

只知道一个仰角，相当于只得到一个方程，未知量却有两个，所以一般无法唯一确定结果。

## 例题 2

为什么实际活动中经常要求“尽量让观测点、目标点处在同一平面内”？

### 讲解

因为我们建模时通常把问题看成平面三角形。

如果这些点实际上偏离同一平面太多，那么：

1. 所画三角形就不是实际几何关系；
2. 角度和距离公式会产生系统误差；
3. 算出的结果即使过程正确，也未必接近真实值。

## 方法总结

测量类探究最重要的不是算得快，而是模型清楚：

1. 哪些量能直接量；
2. 哪些量不能直接量；
3. 间接量由哪个三角形给出；
4. 误差会从哪一步进入。

## 常见错误

1. 没有先设计方案就开始测；
2. 只记计算结果，不记原始数据；
3. 角度单位、长度单位混乱；
4. 忽略地面不水平、目标不共面等实际误差。

## 练习 2

说明为什么基线长度的测量精度会直接影响最终结果。

### 答案

因为基线是后续所有三角形计算的起点，基线一旦有误，后面由定理推出的边长都会整体偏移。

### 讲解

在很多方案中，正弦定理先由基线 $CD$ 推出 $AC,BC$，再由这些边推出目标距离。

所以基线误差会被层层传递，甚至被放大。

### Python 实现

<InlinePythonPlayground
  code={`base = 100
errors = [-1, 0, 1]

for e in errors:
print("基线=", base + e)
`}
/>

## 练习 3

如果测角仪读数误差约为 $1^\circ$，这种误差更可能在哪些情形下被放大？

### 答案

当三角形很扁、目标很远、观测角很小或两个候选位置非常接近时，更容易被放大。

### 讲解

角度变化虽然只有 $1^\circ$，但如果对应的是很长的边，或三角形接近退化，那么边长计算会对角度特别敏感。

### Python 实现

<InlinePythonPlayground
  code={`import math

for angle in [5, 6, 30, 31]:
print(angle, math.sin(math.radians(angle)))
`}
/>

## 练习 4

为什么条件允许时，最好再用其他方式估计一次真实距离做对比？

### 答案

因为这样可以检验模型是否合理，并帮助定位误差来源。

### 讲解

如果计算值和真实值很接近，说明方案有效；如果差距较大，就要回头检查：

1. 是角度测错了；
2. 还是模型假设不成立；
3. 还是计算步骤有误。

### Python 实现

<InlinePythonPlayground
  code={`calc = 32.4
real = 31.8
print("误差=", calc - real)
print("相对误差=", (calc - real) / real)
`}
/>
