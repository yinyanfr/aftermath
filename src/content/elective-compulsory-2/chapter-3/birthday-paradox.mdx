# 生日悖论的解释与模拟

## 学习目标

1. 理解生日悖论到底在说什么；
2. 会从反面事件入手计算概率；
3. 会写出“至少两人生日相同”的概率公式；
4. 体验随机模拟和理论计算之间的对应；
5. 学会区分“与我同生日”和“任意两人同生日”这两类问题。

## 为什么要学这个内容

生日悖论最有意思的地方，是它和人的直觉不一致。它能帮助我们真正理解“概率小不等于不可能，概率大也不一定显眼”。

## 预备知识

1. 基本排列组合；
2. 对立事件概率公式；
3. 简单随机模拟思想。

## 生日悖论说的是什么

在一个有 $n$ 个人的群体中，至少有两个人生日相同的概率，随着 $n$ 的增大上升得很快。

很多人直觉上觉得这个概率不大，但当 $n=23$ 时，这个概率就已经超过 $0.5$。

## 为什么常从反面计算

“至少两人生日相同”直接算很麻烦，因为重复情况很多。

更容易的是先算反面事件“所有人的生日都不同”，再用

$$
$P(\text{match}) = 1 - P(\text{all-distinct})$
$$

## 概率公式

假设一年有 365 天，而且每一天出生的可能性相同。若 $n\le 365$，则

$$
$P(\text{all-distinct})=\frac{A_{365}^n}{365^n}$
$$

因此

$$
$P(\text{match})=1-\frac{A_{365}^n}{365^n}$
$$

## 例题 1

写出 3 个人中至少两人生日相同的概率公式。

### 讲解

$$
P=1-\frac{365\times 364\times 363}{365^3}
$$

## 例题 2

为什么“班里有人和我同生日”的概率，比“班里任意两人同生日”的概率小得多？

### 讲解

因为前者只围绕你这一个人比较，后者允许班里任意两个人配对。可比较的对象对数多得多，所以后者概率大得多。

## 模拟思路

做模拟时，可以把生日看成 1 到 365 的随机整数：

1. 随机生成 $n$ 个整数；
2. 判断是否有重复；
3. 重复很多次；
4. 用“出现重复的次数 ÷ 总试验次数”估计概率。

## 方法总结

1. 重复情况复杂时，优先算对立事件；
2. 理论公式给精确表达，模拟给直观印象；
3. 模拟次数越多，结果一般越接近理论值；
4. 同样是“生日问题”，要先看事件定义。

## 常见错误

1. 把“有人和我同生日”误当成“任意两人同生日”；
2. 直接把重复情况一类类列举，导致漏计；
3. 忘了用补集公式；
4. 模拟次数太少就急着下结论。

## 练习 1

写出 4 个人中所有人生日都不同的概率公式。

### 答案

$$
\frac{365\times 364\times 363\times 362}{365^4}
$$

### 讲解

第 1 人有 365 种生日，第 2 人要与第 1 人不同，所以有 364 种，依次类推。

### Python 实现

<InlinePythonPlayground
  code={`numerator = 365 * 364 * 363 * 362
denominator = 365 ** 4
print(numerator / denominator)`}
/>

## 练习 2

写出 4 个人中至少两人生日相同的概率公式。

### 答案

$$
1-\frac{365\times 364\times 363\times 362}{365^4}
$$

### 讲解

直接用对立事件：至少两人相同 = 1 - 全都不同。

### Python 实现

<InlinePythonPlayground
  code={`p = 1 - (365 * 364 * 363 * 362) / (365 ** 4)
print(p)`}
/>

## 练习 3

解释为什么 366 个人中一定至少有两人生日相同。

### 答案

因为只有 365 个不同日期，而有 366 个人，依据抽屉原理，至少两人落在同一天。

### 讲解

把 365 天看成 365 个抽屉，把 366 个人放进去，至少有一个抽屉里有两个人。

### Python 实现

<InlinePythonPlayground
  code={`days = 365
people = 366
print(people > days)`}
/>

## 练习 4

若随机模拟 10000 次，每次抽取 23 个生日，统计有重复的次数占比，这个占比估计什么？

### 答案

估计 23 人中至少两人生日相同的概率。

### 讲解

频率在大量重复试验下可以用来估计概率，所以这个占比就是对应事件概率的近似值。

### Python 实现

<InlinePythonPlayground
  code={`import random
trials = 1000
success = 0
for _ in range(trials):
    birthdays = [random.randint(1, 365) for _ in range(23)]
    if len(set(birthdays)) < len(birthdays):
        success += 1
print(success / trials)`}
/>

## 练习 5

在生日问题中，为什么说“直觉”不可靠？

### 答案

因为人们容易只盯住某一个人，却忽略了群体中存在大量两两配对的机会。

### 讲解

概率大小常由“可能碰撞的对数”决定。人对这种组合数量增长通常不敏感，所以容易低估概率。

### Python 实现

<InlinePythonPlayground
  code={`n = 23
pairs = n * (n - 1) // 2
print(pairs)`}
/>
